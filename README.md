# Stonia

Stonia（小石之原）是一个轻探索休闲小游戏。玩家掉落到一片被石头覆盖的小地图中，通过凿开石头、收集资源、制造工具、种植、烹饪和阅读遗落书页，慢慢把荒地探索出来。

游戏没有死亡或失败压力，目标是让玩家舒服地探索一会儿：点击、挖掘、获得新东西，再由新东西解锁新的交互。

## 当前玩法概览

- 地图是 11×11 的格子地，玩家从中心开始探索。
- 玩家初始拥有 10 点体力，每次凿石消耗 1 点体力。
- 凿开石头会获得经验和随机矿物，也可能发现植物、水池、动物、宝箱、纸袋、特殊盒子或遗落书页。
- 玩家可以用 `WASD` 或方向键移动，只能在已清开的普通地面上移动，不能穿过石头或水池。
- 资源可以用于制造、铸造、烹饪和建造，例如铸造台、石斧、铁剑、箱子、篝火和庇护所。
- 昼夜由凿石次数推进，夜晚视野会缩小，篝火和水晶球可以提供照明。
- 当前通关目标是收集 7 张遗落书页，并在水晶球处全部提交。
- 当前已实现的系统还包括背包容量与箱子储物、设施放置、放置台等待、种植和小麦/土豆成长、鸡牛狼与良田犬互动、钓鱼挤奶、金银盒子商店、冒险手册、物品/地图图鉴、语言切换和开发者工具。

更完整的玩法规则见：`Stonia-Game-Docs/GameDocs/游戏玩法流程.md`

已实现功能的分系统记录见：`Stonia-Game-Docs/GameDocs/已实现功能系统/已实现功能总览.md`

## 项目结构

- `index.html`：游戏简介页，提供进入 Web 的入口。
- `Stonia.html`：当前可运行的游戏 Web，玩法、界面和交互逻辑主要都在这里。
- `pages/items.html`：物品一览页面，展示当前 Web 物品、获取途径、用途和合成关系。
- `data/items.js`：统一管理物品 id、名称、贴图、分类、游戏内字段和各类展示文案。
- `data/pages.js`：统一管理 7 张遗落书页的标题和正文。
- `data/recipes.js`：统一管理“平台 + 原料 + 产物”的制造、铸造和烹饪配方表。
- `assets/`：游戏资产目录，后续美术、音乐、音效等资源放在这里。
- `assets/2D/`：2D 美术资产建议放置位置，包括像素角色、地形、物品图标、UI 切图等。
- `Stonia-Game-Docs/`：项目文档工作区。
- `Stonia-Game-Docs/GameDocs/`：游戏设计、玩法流程、资产需求等正式游戏文档。
- `Stonia-Game-Docs/GameDocs/美术音乐资产需求.md`：当前美术和音乐资产需求清单。
- `Stonia-Game-Docs/GameDocs/已实现功能系统/`：按系统记录当前已经实现的探索、资源、生态、通关与引导功能。
- `Stonia-Game-Docs/Stand Up/`：站会、进度同步和临时记录。

## 查看与开发

当前 Web 是纯前端单文件实现，可以直接打开根目录下的 `index.html`，再点击入口进入 `Stonia.html` 游玩。

如果使用 Cursor 或 VS Code，可以直接编辑 `Stonia.html` 与 `Stonia-Game-Docs/` 下的文档。修改玩法前，建议先查看 `游戏玩法流程.md`，确认当前规则和已实现内容。

## 贡献指南

### 玩法与代码

- 玩法改动以 `Stonia.html` 的实际实现为准；新增物品、书页或配方优先改 `data/` 下的数据文件。
- 修改玩法后，请同步更新 `Stonia-Game-Docs/GameDocs/游戏玩法流程.md`。
- 如果新增系统、道具、事件或通关条件，也请在 README 中适当补充概览。

### 文档

- 正式设计文档放在 `Stonia-Game-Docs/GameDocs/`。
- 日常想法、站会记录或临时讨论可以放在 `Stonia-Game-Docs/Stand Up/` 或其他文档子目录中。
- 文档尽量写清楚“当前已实现”和“未来设想”的区别。

### 美术、音乐与音效

- 美术资产统一放在 `assets/` 下，2D 像素资源优先放在 `assets/2D/`。
- 新增资产前，建议先查看 `Stonia-Game-Docs/GameDocs/美术音乐资产需求.md`。
- 当前期望美术风格是类似《星露谷物语》的 2D 像素风，整体色调偏暖。
- UI 设计需要同时考虑手机竖屏和 PC 横屏两版。
