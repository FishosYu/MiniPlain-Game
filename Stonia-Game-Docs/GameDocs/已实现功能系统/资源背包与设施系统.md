# 资源背包与设施系统

本文档记录当前 `Stonia.html` 中资源获得、背包容量、箱子储物、制造/铸造/加工平台和设施放置规则。完整配方表见 [[配方详细]] 与 [[烹饪详细]]。

## 1. 物品数据

物品定义集中在 `data/items.js`，游戏运行时通过 `window.MINIPLAIN_ITEMS` 注入。

物品字段主要包括：

- `name`：显示名称。
- `vol`：体积；金币、银币等可为 0。
- `category`：图鉴与页面展示分类。
- `use`：直接使用效果，例如回复体力。
- `place`：可放置物对应的地图结构。
- `plant`：是否可作为植物类交互使用。
- `texts`：页面、图鉴、获取途径和用途说明。

当前分类：

- 基础资源
- 食材与种植
- 动物产物
- 工具与装备
- 设施与建造
- 货币与任务
- 加工材料
- 料理

主要实现：

- `ITEM_DATA`
- `ITEMS`
- `CODEX_ITEM_CATEGORIES`
- `discoverItem(id)`

## 2. 背包容量

### 2.1 背包规则

- 背包容量上限为 30 单位。
- 物品按 `vol * count` 占用容量。
- 大多数物品体积为 1。
- 金币、银币和鱼者等特殊物品不占容量。
- 添加物品时，如果容量不足会失败或把奖励留在地面。

主要实现：

- `INV_MAX`
- `inventory`
- `containerVolume(list)`
- `invVolume()`
- `addItem(id, n)`
- `addToContainer(list, maxVol, id, n, fullMsg)`

### 2.2 地面掉落

当木宝箱、纸袋等奖励无法放入背包时，奖励会以 `groundLoot` 形式留在当前格。玩家之后可以点击该格拾取。

主要实现：

- `giveItemOrDropOnCell(x, y, id, n)`
- `dropLootOnCell(x, y, id, n)`
- `interactCell(x, y)`

## 3. 箱子储物

箱子通过工具箱制造获得，放置后变为地图结构 `storage_chest`。

规则：

- 箱子容量为 30 单位。
- 玩家站在箱子格上时，右侧显示箱子面板。
- 背包与箱子之间支持拖拽转移。
- 支持单个转移和整组转移。
- `Shift + 点击` 或长按可整组转移。
- 每个箱子以坐标作为存储 key，互不共享内容。

主要实现：

- `CHEST_MAX`
- `chestStorages`
- `chestStorageKey(x, y)`
- `getChestStorageAt(x, y)`
- `getActiveChestStorage()`
- `transferStackItem()`
- `transferEntireStack()`
- `setupChestBulkTransfer()`
- `renderChest()`

## 4. 制造、铸造与加工平台

### 4.1 配方数据

所有制造、铸造、烹饪和加工配方集中在 `data/recipes.js` 的 `window.MINIPLAIN_RECIPES`。

配方结构：

- `station`：平台类型。
- `inputs`：输入材料，顺序无关。
- `output`：产物。
- `count`：产物数量，默认为 1。
- `returns`：制作后返还的工具，例如锤子和石斧。

当前平台：

| 平台 | station | 入口 | 功能 |
|------|---------|------|------|
| 破旧工具箱 | `toolbox` | 站在或邻近工具箱 | 制造基础设施和加工材料 |
| 铸造台 | `forge` | 站在或邻近铸造台 | 铸造工具、锅具、金银币和板材 |
| 篝火 | `campfire` | 靠近篝火且背包有铁锅 | 基础料理 |
| 铁板 | `iron_plate` | 站在铁板上 | 煎烤类料理 |
| 石磨 | `mill` | 站在石磨上 | 小麦磨面粉 |
| 料理台 | `prep_table` | 站在料理台上 | 面团、蛋糕胚、汉堡组装 |
| 烤箱 | `oven` | 站在烤箱上 | 面包、馒头、包子、蛋糕 |

主要实现：

- `RECIPES`
- `COOK_STATIONS`
- `CRAFT_STATIONS`
- `RECIPE_LOOKUP`
- `recipeKey(a, b)`
- `recipeForStation(station, a, b)`

### 4.2 模式选择

合成区始终是两个材料格加一个结果格。系统根据玩家所在位置和材料判断当前模式：

- 在工具箱附近时为制造。
- 在铸造台附近时为铸造。
- 在非篝火料理平台上时为烹饪/加工。
- 在篝火附近时，必须背包中有铁锅才进入烹饪。
- 如果材料与当前平台不匹配，会提示需要站在对应平台。

主要实现：

- `currentRecipeStation()`
- `getCraftMode()`
- `slotsHaveFood()`
- `slotsHaveCraftMat()`
- `canCookHere()`

### 4.3 制作结算

制作时：

- 系统用材料 id 排序后的 key 查找配方，因此两个材料顺序无关。
- 单材料配方只需要一个材料格。
- 产物必须能放入背包，否则制作取消并返还材料。
- 配方带 `returns` 时，会把工具返还到背包。
- 做出料理类物品时会解锁冒险手册中的第一份料理成就。

主要实现：

- `resolveRecipeEntry(a, b, mode)`
- `resolveRecipe(a, b, mode)`
- `doCraft()`
- `packRecipeResult(recipe)`
- `unpackResult(result)`

## 6. 设施放置

### 6.1 通用放置流程

可放置物在背包中点击后进入放置模式，地图上符合条件的格子会高亮。再次点击有效格后消耗物品并放置结构。

通用条件：

- 目标格已经清开。
- 目标格是普通地面。
- 目标格没有植物、结构、生物和地面掉落。

主要实现：

- `placedMode`
- `isPlaceableItem(id)`
- `togglePlacedMode(mode, activateLog)`
- `canPlaceOnCell(x, y)`
- `tryPlaceStructure(x, y)`

### 6.2 已实现设施与限制

| 设施 | 背包物品 | 地图结构 | 放置限制 |
|------|----------|----------|----------|
| 铸造台 | `forge` | `forge` | 普通空地 |
| 箱子 | `chest` | `storage_chest` | 普通空地 |
| 篝火 | `campfire` | `campfire` | 普通空地，不能贴近八方向水池 |
| 烤箱 | `oven` | `oven` | 普通空地 |
| 铁板 | `iron_plate` | `iron_plate` | 普通空地，必须上下左右邻接篝火 |
| 石磨 | `mill` | `mill` | 普通空地 |
| 料理台 | `prep_table` | `prep_table` | 普通空地 |
| 放置台 | `aging_table` | `aging_table` | 普通空地 |
| 桥 | `bridge` | `bridge` | 只能放在无阻挡水池格 |
| 庇护所 | `building` x4 | `shelter` | 8 级后，在 2×2 普通空地建造 |

特殊判断：

- 篝火使用 `hasWaterNeighbor8(x, y)` 禁止贴水。
- 铁板使用 `canPlaceIronPlate(x, y)` 检查是否邻接篝火。
- 桥使用 `canPlaceBridge(x, y)` 确保目标是水面且没有阻挡。
- 庇护所使用 `canPlaceBuildingAt(x, y)` 检查 2×2 空地。
- 放置台一次只能放入一格物品；牛奶、水果、土豆、水和小麦会随时间转换。持有鱼者时，牛奶和水果不会转换成失败产物。

## 7. 庇护所

庇护所是当前等级系统带来的建筑目标。

建造条件：

- 玩家等级至少 8。
- 背包中有 4 个建材。
- 点击一个可容纳 2×2 的空地左上角。
- 四格都必须是普通空地，且没有植物、结构、生物或掉落物。

建造后：

- 四个格子都变为 `shelter`。
- 点击庇护所可以休整。
- 休整会推进凿石次数和时间相关效果，但不消耗体力、不获得矿物和经验。

主要实现：

- `placedMode === 'building'`
- `canPlaceBuildingAt(x, y)`
- `interactCell(x, y)`

## 8. 货币加工

当前货币来源和用途已经打通：

- 金块可在铸造台变为 2 金币。
- 银块可在铸造台变为 2 银币。
- 金币用于金色盒子商店。
- 银币和银块折算余额用于银色盒子料理商店。

主要实现：

- `getSilverBalance()`
- `spendSilver(amount)`
- `openGoldenMerchant()`
- `openSilverMerchant()`
