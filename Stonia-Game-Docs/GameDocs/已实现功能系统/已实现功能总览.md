# 已实现功能总览

本文档组记录当前 `Stonia.html` 已经实现并可游玩的功能系统。数值、概率、入口和命名以当前 Web 版本为准；配方细节仍以 [[配方详细]] 与 [[烹饪详细]] 为主。

## 1. 文档范围

本目录用于回答“游戏现在已经做到了什么”。其中：

- [[探索地图与发现系统]]：地图、凿石、随机发现、昼夜、视野和通关关键物。
- [[资源背包与设施系统]]：物品容量、箱子储物、制造/铸造/放置台平台、设施放置和建筑。
- [[种植动物与商店系统]]：植物成长、种植、小麦、土豆、鸡牛狼、良田犬、钓鱼、挤奶、金银商店。
- [[书页通关与引导系统]]：遗落书页、水晶球、胜利流程、冒险手册、物品/地图图鉴、语言切换和开发者工具。

## 2. 当前已实现的核心循环

当前游戏的主循环已经形成：

1. 玩家从 11×11 石头地图中心开始。
2. 点击相邻石头，消耗体力凿开地图。
3. 凿石获得经验、矿物和石头下的隐藏内容。
4. 隐藏内容提供植物、动物、工具、设施、商店、书页和水晶球。
5. 资源进入背包后，可用于制造、铸造、烹饪、放置等待、种植、建造和交易。
6. 食物和升级补充体力，推动更深探索。
7. 收集 7 张遗落书页，在水晶球处全部提交后通关。

## 4. 已形成的系统联动

### 4.1 探索驱动资源

凿石既是地图开拓方式，也是时间推进器。它会同时影响体力消耗、经验增长、矿物掉落、隐藏内容揭示、昼夜变化、植物成长、鸡蛋孵化和动物产蛋。

### 4.2 资源驱动设施

铁块、石头、木材、金银等基础资源进入背包后，可以通过工具箱与铸造台做出关键设施和工具。设施再解锁新的平台，例如篝火解锁料理，铁板、石磨、料理台和烤箱解锁更复杂的食物链。

### 4.3 生态驱动料理

植物、动物、钓鱼、种植和放置台共同提供食材与加工产物。料理可以恢复体力，放置台还能把牛奶、水果、土豆、水和小麦转化为酸奶、果酒、发芽土豆、干草等时间产物。

### 4.4 目标驱动收束

遗落书页和水晶球把随机探索收束成明确目标。书页有进度保底，水晶球有后期出现概率和兜底机制，避免通关目标完全卡在随机数上。

### 4.5 引导驱动学习

冒险手册用章节和成就引导玩家从“找到工具箱”逐步走到“时间的配方”和“提交书页”。图鉴则记录已发现物品与地图发现，帮助玩家回看物品用途和地图内容。

## 5. 当前仍需注意的实现边界

- 当前没有完整存档/读档流程，除图鉴发现记录外，大局进度不会持久化。
- `wolf_fur`、`fruit_battery` 等物品已经进入数据和图鉴，但玩法用途仍较少。
- 部分早期文档中的烹饪简表已落后于 `data/recipes.js` 的完整配方；新增或核对配方时优先查看数据文件。
- 代码仍主要集中在 `Stonia.html`，文档中提到的函数名用于定位当前实现，不代表已经模块化拆分。
