# MiniPlain 美术 / 音乐资产需求

本文档基于当前已实现玩法整理资产需求。现阶段优先列出美术需求，音乐与音效需求后续可在玩法节奏进一步稳定后补充。

## 1. 整体美术方向

目标风格：

- 2D 像素风格，参考《星露谷物语》的温暖、轻松、手作感。
- 色调以暖色为主，整体偏米色、土黄、浅棕、暖绿，避免过冷、过暗或高压氛围。
- 游戏体验应传达“独自在石头地中慢慢探索、逐渐恢复生机”的感觉。
- 像素颗粒感清晰，但不要过度复古到难以识别。
- 地图、角色、UI 图标需要保持同一套像素密度和描边规则。

建议规格：

- 地图格子基础素材建议按 32×32 或 48×48 像素制作，后续可按屏幕缩放。
- UI 图标建议提供 32×32、64×64 两档，便于背包、合成栏、弹窗中复用。
- 所有可交互对象需要有清晰轮廓，保证手机小屏也能识别。

## 2. 界面设计需求

当前游戏需要同时考虑两套主要布局：

- 手机竖屏版：单手点击、信息分层、地图优先。
- PC 横屏版：地图与背包/合成面板并列，操作信息常驻。

### 2.1 手机竖屏界面

手机竖屏版重点是减少拥挤，保证地图点击准确。

需要设计：

- 顶部状态栏：体力、等级、经验、凿石次数、昼夜状态。
- 主地图区域：11×11 地图可完整显示，格子点击区域必须足够大。
- 底部快捷区域：背包入口、合成入口、日志入口、当前选中物品提示。
- 背包抽屉：从底部或侧边弹出，显示物品格、容量、书页。
- 合成/铸造/烹饪面板：两个材料格、一个结果格、确认按钮。
- 地图交互提示：凿石高亮、可放置高亮、夜晚视野遮罩。
- 弹窗：宝箱、纸袋、金色盒子、银色盒子、水晶球、通关界面。
- 日志提示：短消息浮层或底部滚动消息，避免长期占用地图空间。

#### 手机版 UI 设计重点：

- 地图格子最小点击区域建议不低于 40×40 逻辑像素。
- 背包物品图标需要足够大，物品数量与名称可分层显示。
- 合成操作应支持点击选择，不依赖拖拽。
- 弹窗按钮需要适合触屏，主按钮和关闭按钮区分明显。
- 夜晚时遮罩不能让玩家完全失去方向感，需要保留已知地图结构的弱提示。

### 2.2 PC 横屏界面（先放着不用干）

PC 横屏版可以保留更多常驻信息，强调地图与管理面板并列。

需要设计：

- 左侧或中间主地图区域。
- 右侧信息面板：背包、箱子、合成区、操作提示。
- 顶部或地图上方状态栏。
- 日志区域：显示最近探索、获得物、失败提示。
- 弹窗层：商店、书页阅读、水晶球提交、通关。
- 鼠标悬停提示：格子信息、物品用途、合成提示。

PC 版 UI 设计重点：

- 地图与右侧面板视觉权重需要平衡，避免背包抢走探索焦点。
- 合成区需要明确“制造 / 铸造 / 烹饪”三种状态。
- 鼠标 hover、可点击、高亮、不可用状态要有统一视觉规范。
- 箱子面板出现时，需要清楚区分背包与箱子两个容器。

## 9. 美术优先级

第一优先级：可玩 Demo 替换表情符号

- 地图格子：石头、普通地面、水池、夜晚遮罩。
- 玩家基础形象与移动方向。
- 背包格、按钮、状态栏、弹窗。
- 主要物品图标：石头、铁块、木材、水果、树苗、金币、书页。
- 关键结构：工具箱、铸造台、箱子、篝火、水晶球。
- 凿石、拾取、升级、背包满、体力不足等基础反馈。

第二优先级：扩展生态与生产链

- 鸡、牛、狼。
- 水果、蔬菜、灌木、树、小麦。
- 纸袋、木宝箱、金色盒子、银色盒子。
- 鱼竿、铁剑、石斧、铁锅、料理图标。
- 种植、砍树、烹饪、钓鱼相关反馈。

第三优先级：氛围与完成度

- 地面与石头多变体。
- 更细的昼夜过渡。
- 篝火、水池、水晶球循环动画。
- 庇护所完整 2×2 建筑。
- 通关界面插画或特殊动画。

## 10. 暂定音乐与音效方向

音乐需求暂不展开制作清单，先记录方向：

- 整体音乐应轻松、温暖、低压力，适合长时间短循环。
- 白天音乐偏木吉他、钟琴、轻打击或温暖合成器。
- 夜晚音乐更安静，但不恐怖。
- 随着探索深入音乐旋律可以更加丰富。
- 通关音乐需要有小规模庆祝感。

- 种植
- 砍树
- 动物叫声
- 昼夜切换
